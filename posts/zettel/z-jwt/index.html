JWT 验证

📅 2023-04-02 | ✒️ 694 | ⌛ 2分钟 | 📁 编程  | 🏷️ 
    web

JWT 是一种网站登录状态的验证方式，服务端通过验证特定 token 来判断用户是否登录、访问权限等状态。主要特点是不依赖服务端状态，可以跨域验证。一个 JWT 格式的 token 包含 Header、Payload、Signature 三个部分，服务端使用密钥对 Header 和 Payload 进行加密生成 Signature，因此可以确保 token 不被篡改，并且在各个服务器上都能验证。

之前验证用户登录状态是通过客户端 Cookie 和服务器 Session 共同维护，有以下几步：

  1. 客户端发送用户名密码给服务端进行登录；
  2. 服务端登陆验证通过后将 SessionID 作为 token 给客户端；
  3. 客户端将 token 保存在 Cookie 中；
  4. 客户端访问网页时需要发送 token；
  5. 服务端验证 token 是存在的 Session，则可以为客户端提供服务；

使用 Session 不能在多个服务器上提供验证服务，用户在 A 服务器上登录后访问同一公司 B 服务器上的业务，就需要重新登录，即不支持跨域登录信息的维持。

利用 JWT 可以将验证与 Session 分离，不受服务器限制，从而支持跨域登录信息的维持。

一个 JWT 类型的 token 类似Header.Payload.Signature，由.连接三部分，具体介绍如下：

  • Header：一个 JSON 对象，包含 token 类型和加密算法，使用 Base64URL 生成上面 token 中的 Header；
  • Payload：一个 JSON 对象，包含业务所需的字段，例如username，不适合放敏感数据，使用 Base64URL 生成上面 token 中的 Payload；
  • Signature：将 Base64URL 的 Header 和 Payload 以逗号相连，再使用服务端的密钥和 Header 中的加密方法加密，对 Header 和 Payload 生成签名，防止被篡改；

在客户端登录后，服务端生成 token 发给客户端，客户端保存在 Cookie 中，每次访问网页时有两种方法携带 token：

  1. 使用 POST 将 token 作为参数传递；
  2. 放在 HTTP 请求头的Authorization域中，确保跨域的时候都能携带；
