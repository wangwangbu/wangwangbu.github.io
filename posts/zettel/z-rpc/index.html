RPC 远程过程调用

📅 2023-04-18 | ✒️ 189 | ⌛ 1分钟 | 📁 编程  | 🏷️ 
    RPC

远程过程调用（Remote Procedure Call, RPC）是一种通讯协议，用于请求服务，包含传输协议和序列化协议。

和 HTTP 的区别：

  • HTTP 是一种传输协议，而 RPC 是一种通讯方式，可以使用 HTTP 也可以使用 TCP 等，像 gRPC 使用的是 HTTP 2.0。
  • HTTP 一般用于 B/S 架构，而 RPC 多用于 C/S 架构，RPC 一直是分布式系统通信的标配，现在因为融合，S 使用的就是 HTTP+RPC 的混合架构。
  • 现在的 RPC 更可以看作是建立在 HTTP 上具有多种封装特性的协议，主要是为了使用其特性。
